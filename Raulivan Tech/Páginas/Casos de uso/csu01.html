Sumário: A unidade de saúde usa o sistema para registrar uma vacinação.

Ator primário: Unidade de saúde.

Ator secundário: Paciente

Fluxo principal

  1. O paciente cadastrado (conforme a RN 03) solicita para a unidade de saúde a vacinação (Conforme a RN05).
  2. A unidade de saúde deverá informar ao sistema os dados do paciente e qual vacina será aplicada.
  3. O sistema deverá dar baixa da vacina especificada no estoque.
  4. O sistema deverá registrar a vacinação na carteira de vacinação do paciente.
  5. A unidade de saúde deverá fornecer ao sistema uma confirmação após realizar a vacinação do paciente.
  6. O sistema deverá fornecer um comprovante de vacinação após a confirmação (Conforme a RN06).

Fluxo de Excecção (1): Violação da de RN03 - Paciente solicitou vacinação sem estar cadastrado no sistema.

  • O sistema informa que o paciente não está cadastrado e sugere o cadastro do mesmo.
  • Caso aceite a sugestão o sistema será direcionado para CSU00.

Fluxo de Excecção (2): Violação da RN02 - Foi selecionado para uma vacina indisponível no estoque.

  • O sistema irá informar que a vacina selecionada não está disponível e impedirá o fluxo principal.

Pós condições: A vacinação foi realizada e registrada no sistema.

Regras de negócio: RN02, RN03, RN05, RN06
