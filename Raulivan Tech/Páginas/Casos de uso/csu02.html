Sumário: Paciente faz o agendamento para tomar uma vacina

Ator primário: Paciente.

  1. Paciente solicita um agendamento.
  2. O sistema verifica quais vacinas não foram tomadas pelo usuário e as libera para seleção (conforme a RN01)
  3. O cliente seleciona um dos agendamentos que estão disponíveis.
  4. O sistema verifica se há estoque para atender esse agendamento (conforme a RN02).
  5. O cliente seleciona um horário para a vacinação.
  6. O sistema verifica se este usuário tem alguma outra vacina agendada para o mesmo horário (conforme a RN00).
  7. O sistema exibe um resumo do agendamento.
  8. O usuário confirma o agendamento.

Fluxo Alternativo (2): Paciente já tomou todas vacinas

  • O sistema finaliza o processo.

Fluxo Alternativo (4): Não há vacinas

  • O sistema finaliza o processo.

Fluxo Alternativo (6):

  • O sistema encaminha para a seleção de horários.

Pós-condições: O agendamento foi homologado e salvo no calendário do cliente.

Regras de negócio: RN01, RN02, RN00
