Sumário: O administrador poderá cadastrar outros usuários como administrador.

Ator primário: Administrador.

Ator secundário: Outro usuário.

Pré-condições: O administrador esteja cadastrado no sistema e ter pelo menos um outro usuário no sistema.

Fluxo principal

  1. O administrador solicita o cadastro de mais um administrador;
  2. O sistema pede as informações de e-mail da pessoa;
  3. O administrador coloca as informações do usuário;
  4. O sistema valida o e-mail, se esse usuário existe e que ainda não é administrador;
  5. O administrador confirma operação;
  6. O sistema modifica o usuário como novo administrador do sistema e finaliza o caso de uso.

Fluxo Alternativo (2 ou 3 ou 4): Usuário desiste de tornar um outro usuário como administrador.

  1. O administrador clica no botão cancelar e o caso de uso termina.

Fluxo Alternativo (3): O usuário indicado pelo administrador não existe.

  1. Retorna ao passo 3 e o sistema solicita novamente que o administrador coloque um usuário válido.

Pós-condições: O sistema fornece uma nota notificando a atribuição de administrador para o usuário indicado.
