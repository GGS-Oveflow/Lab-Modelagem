Sumário: O cliente utiliza o sistema para se cadastrar

Ator primário: Cliente

Ator secundário: Administrador

Pré-condições: O cliente deve ter aberto a tela de login

Fluxo principal

  1. O cliente solicita o cadastro;
  2. O sistema exibe um formulário para capturar os dados;
  3. O cliente preenche todas as informações requisitadas;
  4. O cliente aperta o botão de cadastrar;
  5. O sistema verifica os dados e envia e confirma o cadastro.

Fluxo Alternativo (3): Caso todos os dados não sejam preenchidos

  1. O sistema exibe o item obrigatório e exige que ele seja preenchido;
  2. O cliente preenche o item faltante;
  3. O cliente aperta o botão de cadastrar;
  4. O sistema verifica os dados e envia e confirma o cadastro.

Fluxo Alternativo (5): A verificação indicou algum erro

  1. O sistema limpa a caixa de texto incorreta e informa o erro, sugerindo a maneira como o deve preenche-la;
  2. O cliente preenche da maneira sugerida;
  3. O cliente aperta o botão de cadastrar;
  4. O sistema verifica os dados e envia e confirma o cadastro.

Pós-condições: Os dados são armazenados no banco de dados e o usuário é redirecionado para a homepage.
